# Run output

### Finding 1: MonetaryValue rename imposes currency semantic on configurable Value

- **Severity:** Block
- **Evidence:** src/Umbraco.Engage.AI/Tools/GetGoalsTool.cs — `decimal MonetaryValue` in `GoalResult` record (renamed from generic `Value`)
- **Framework citation:** Q3 generic-vs-imposed semantic, Q1(c) Stone-vs-Opinion (Opinion treated as Stone)
- **Issue:** Goal.Value in Engage Core is a generic decimal the marketer configures per goal — it may carry monetary value, a raw count, a priority score, a weight, or zero for tracking-only goals. The rename to MonetaryValue makes the wire-format field claim "this is money." The data layer does not back that claim. An LLM consuming the response will report findings as dollar amounts and will fabricate currency totals when marketers have configured Values as a mix of money, scores, and weights.
- **Recommendation:** Revert the field name to `Value`. Update the tool description to surface configurability: *"Value is a marketer-configured decimal whose meaning depends on goal setup — monetary, count, score, weight, or zero for tracking-only. Do not assume currency."* Longer-term, add a field-level `SemanticClass: Stone | Opinion` enum to the result record (see engage-design-patterns.md §2) so future AI reviewers cannot silently promote Opinion fields to Stone via rename.

### Finding 2: Tool description repeats the monetary claim in prose

- **Severity:** Concern
- **Evidence:** src/Umbraco.Engage.AI/Tools/GetGoalsTool.cs — `Description`: `"Returns each Engage goal with its name, goal type, monetary value, whether it is the main goal, and whether it is currently active."`
- **Framework citation:** Q3 generic-vs-imposed semantic
- **Issue:** The Description string handed to the LLM names the field "monetary value" in plain prose. Even if the wire-format symbol is corrected, this sentence still instructs the model to narrate the decimal as currency. Description text is part of the wire format the LLM repeats; it imposes the same Stone-class money semantic on an Opinion-class configurable decimal.
- **Recommendation:** Drop "monetary" from the Description. Replace the clause with something semantic-neutral that flags configurability, e.g. *"…with its name, goal type, configured value (a marketer-set decimal — may be monetary, a count, a score, a weight, or zero for tracking-only goals; do not assume currency), whether it is the main goal, and whether it is currently active."* Keep the prose and the field name aligned on the generic reading.
