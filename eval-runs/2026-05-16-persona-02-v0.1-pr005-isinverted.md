- **Severity:** Block

- **Framework citation:** Q2 polarity semantic
- **Issue:** A marketer asking "which of our goals look healthiest right now?" will be told their `Cart Abandoned` or `Bounced` goal is a configured KPI sitting active in the system, and the moment that result is composed with any completion-count tool the LLM will rank those failure-mode goals alongside `Newsletter Signup` and `Purchase` as if more = better. The wire format strips `IsInverted` before it ever reaches the model, so the LLM has no way to flip the arrow on goals where high counts mean customers are leaking out of the funnel. Whichever inverted goal racks up the most "completions" this quarter gets narrated up the chain as a win against a metric that is actually the marketer's worst failure mode.
- **Recommendation:** Add `IsInverted` to `GoalResult` directly off `goal.IsInverted` — it is already on `IGoal`, so this is a one-line surface, not new infrastructure. Then add a sentence to the tool description that teaches the LLM the direction-flip: *"Each goal carries an `IsInverted` flag. `IsInverted=true` means a high completion count is BAD — the goal counts a failure mode (cart abandonment, bounce, form-abandon). Do not rank inverted goals against normal goals by raw count, and do not describe a high inverted-goal count as a positive result."* Without both halves (field + description), the LLM either doesn't see the flag or sees it and doesn't know what to do with it.

- **Evidence:** `src/Umbraco.Engage.AI/Tools/GetGoalsTool.cs:23-28` — `Description => "Use this when the user asks what goals are configured or which KPIs are being tracked. ...
