### Finding 1: `MonetaryValue` rename promotes configurable decimal to currency

- **Severity:** Block
- **Evidence:** `src/Umbraco.Engage.AI/Tools/GetGoalsTool.cs:39` — `MonetaryValue: goal.Value` (and `GoalResult` field `decimal MonetaryValue`)
- **Framework citation:** Q3 generic-vs-imposed semantic, Q1(c) Stone-vs-Opinion (Opinion treated as Stone)
- **Issue:** This rename teaches the LLM to call configurable decimals "money." Marketers configure `Goal.Value` as whatever fits their setup — currency for revenue goals, raw counts for volume goals, priority scores for ranking, weights for scoring models, zero for tracking-only — and the data layer enforces none of it. When the LLM totals the response and tells a marketer "your active goals represent $1,840 in monetary value this quarter," it is fabricating dollar arithmetic over a column that mixes dollars, scores, and weights. The marketer will commit budget, forecast pipeline, or brief a CFO against a number that does not exist.
- **Recommendation:** Rename the wire-format field back to `Value` — generic decimal, no unit claim. Then earn the clarity in the tool `Description` instead of the field name: add a sentence like *"Value is a marketer-configured decimal whose meaning depends on each goal's setup — it may be a monetary amount, a count, a priority score, a weight, or zero for tracking-only goals. Do not assume currency; ask the user how their goals are configured before aggregating or labeling units."* That keeps the schema honest and gives the LLM the disclaimer it needs to narrate responsibly.

### Finding 2: Description phrase "monetary value" reinforces the currency claim

- **Severity:** Block
- **Evidence:** `src/Umbraco.Engage.AI/Tools/GetGoalsTool.cs:24` — `"Returns each Engage goal with its name, goal type, monetary value, whether it is the main goal, and whether it is currently active."`
- **Framework citation:** Q3 generic-vs-imposed semantic
- **Issue:** Even with the field renamed, this sentence alone teaches the LLM that every goal has a "monetary value" full stop — no hedge, no configurability note. The LLM will quote this description back to marketers as "I can show you each goal's monetary value" and then narrate non-monetary configurations (scores, weights, counts) as dollars because the tool itself promised money. The underlying `IGoal.Value` is a generic decimal with no currency enforcement; the description is asserting a semantic the data layer never agreed to. Marketers reading goal summaries will read currency where there is none and plan spend accordingly.
- **Recommendation:** Replace "monetary value" with a generic noun and a configurability disclaimer in the same sentence. Something like: *"Returns each Engage goal with its name, goal type, configured value (a marketer-set decimal — may be currency, count, score, weight, or zero; do not assume units), whether it is the main goal, and whether it is currently active."* Putting the disclaimer inline — not in a separate paragraph the LLM may skip — is what stops the narration drift.
